Algorithm Performance Space

Algorithm Performance Space (APS) is a tool to help researchers pick better, more diverse datasets for testing recommender systems algorithms. Instead of selecting popular datasets blindly, APS maps datasets based on how different algorithms perform on them. By using this tool, you can see which datasets are very similar and might not add much new information for training and benchmarking. The graph below visualizes this by reducing performance results into two dimensions using Principal Component Analysis (PCA) based on the evaluations of a selected performance metric such as nDCG, Hit Ratio or Recall with a K-value.
Similarity between datasets is then determined by how close their points are in this two-dimensional performance space using Euclidean distance:
distance = √((x₂ - x₁)² + (y₂ - y₁)²)
The closer two datasets are, the more similar their algorithm performance results. However, each point also has associated variance values that indicate the uncertainty or spread in its position. Instead of relying only on Euclidean distance, similarity is measured based on whether the distance between datasets falls within their combined variance (ellipses in the graph).
combined variance = √(varX₁² + varY₁² + varX₂² + varY₂²)
The smaller the distance relative to the combined variance, the higher the confidence in their similarity. Confidence levels are then calculated and used to display similar datasets.
confidence = 1 - (distance / combinedVariance)

Similarity Confidence Levels:
Very High (>= 0.9) High (0.75 - 0.9) Moderate (0.5 - 0.75) Low (0.25 - 0.5) Very Low (<= 0.25)
